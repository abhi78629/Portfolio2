What is this project?

The primary goal was to develop a diagnostic support tool that predicts whether a patient has diabetes based on diagnostic measurements. We utilized the Pima Indians Diabetes Database, a classic yet challenging dataset in the ML community.

The core challenge resided in the imbalanced nature of the dataset and the presence of missing values (e.g., zero in non-zero features like insulin and BMI). Our approach prioritizes a rigorous preprocessing stage over brute-force modeling.

Development Journey

MILESTONE 1

Initial Baseline

Raw data training with basic KNN. Accuracy was low due to unscaled features and missing zeros.

MILESTONE 2

Data Cleaning

Imputed missing values with median and performed Z-score normalization. Accuracy improved significantly.

MILESTONE 3

SMOTE Integration

Balanced class labels using Synthetic Minority Oversampling to fix recall issues in positive cases.

MILESTONE 4

Hyperparameter Opt.

Grid search for optimal k value and weight metrics. Found k=11 to be a local sweet spot.

Technical Implementation

Data Preprocessing

Handling missing values through iterative imputer and ensuring all numerical inputs are on the same scale for distance calculation.

SMOTE Oversampling

Synthetically balancing the dataset to prevent the model from becoming biased towards non-diabetic predictions.

Hyperparameter Tuning

Executing cross-validated grid searches across k-values, distance metrics (Manhattan vs. Euclidean), and weighting schemes.

Model Evaluation

Moving beyond simple accuracy to focus on Precision/Recall scores and ROC-AUC for clinical relevance.

Personal Reflections

  • Data is King: No amount of model tuning could fix what proper imputation and scaling achieved in v2.0.
  • The 'k' Paradox: I learned that a higher 'k' isn't always better. It's a delicate balance between noise filtering and detail preservation.
  • Clinical Nuance: In healthcare, False Negatives are more dangerous than False Positives. This shifted our optimization focus toward Recall.
